Tipos de paginas web!

Página Estática

Son páginas web básicas que muestran información fija y no cambian según la interacción del usuario.

Página Dinámica

Son páginas web que pueden cambiar el contenido en función de las acciones del usuario, la información de la base de datos u otros factores.

Perfiles de Programadores

1. Programador Frontend

El programador frontend se encarga del desarrollo de la interfaz de usuario de una aplicación web. Utiliza principalmente HTML, CSS y JavaScript.

2. Programador Backend

El programador backend se concentra en el desarrollo de la lógica de la aplicación que se ejecuta en el servidor. Suele utilizar lenguajes como Python, Java, PHP o Node.js.

3. Full Stack Developer

El full stack developer es capaz de trabajar tanto en el frontend como en el backend de una aplicación. Tiene conocimientos en una amplia variedad de tecnologías.

HTML

    "HyperText Markup Language" (Lenguaje de Marcado de Hipertexto), es el lenguaje estándar utilizado para crear páginas web. Es un lenguaje de marcado que define la estructura y el contenido de una página web mediante etiquetas y elementos. Estas etiquetas permiten al navegador web interpretar y mostrar el contenido de manera adecuada al usuario. En HTML, las etiquetas se utilizan para marcar diferentes partes del contenido, como encabezados, párrafos, imágenes, enlaces, formularios, listas y más. Cada etiqueta tiene una función específica y puede tener atributos que proporcionan información adicional sobre el elemento.

Estructura del Código HTML

    DOCTYPE hTML:Es la etiqueta que define que el documento es HTML.

    html: Este elemento envuelve todo el contenido de la página y define que el documento es un documento HTML.

    head: Esta sección contiene metadatos sobre el documento, No se muestra en la ventana del navegador.

    meta charset="UTF-8": Esta etiqueta especifica la codificación de caracteres utilizada en el documento. UTF-8 es la codificación recomendada y es compatible con una amplia gama de caracteres de diferentes idiomas.

    title: Define el título de la página que se muestra en la pestaña del navegador o en la barra de título de la ventana del navegador.

    body: Esta sección contiene todo el contenido visible de la página web, como texto, imágenes, enlaces, formularios, etc. Todo lo que se muestra en el navegador se coloca dentro de este elemento

Etiquetas HTML Más Usadas

CSS

CSS es un lenguaje de estilo utilizado para definir y el formato de un documento HTML o XML. Permite controlar el diseño, el color, la tipografía y otros aspectos visuales de una página web.

Mejora de la Legibilidad con CSS

Selección de Fuentes

Elige fuentes legibles y adecuadas para el contenido. Utiliza la propiedad code font-family code para especificar las fuentes en tu página.

Tamaño del Texto

Ajusta el tamaño del texto para que sea cómodo de leer. Utiliza la propiedad code fontsize code para especificar el tamaño del texto.

Espaciado entre Líneas

El espaciado entre líneas adecuado puede mejorar la legibilidad. Utiliza la propiedad code line-height code para controlar el espaciado entre líneas.

JavaScript

Es un lenguaje de programación que se puede usar para añadir características interactivas en un sitio web. Por ejmeplo: juegos, eventos que ocurren cuando los botones son presionados o los datos son introducidos en los formularios, efectos de estilo dinámicos, animación y mucho más, Javascript por sí solo es bastante compacto aunque muy flexible.

SCRUM

<

  1. Es un marco de administración que los equipos utilizan para organizarse por cuenta propia y trabajar en aras de alcanzar un objetivo común. Describe un conjunto de reuniones, herramientas y funciones para areas de alcanzar un objetivo en común. Al igual que un equipo deportivo que practica para un importante partido, las prácticas de Scrum permiten a los equipos de trabajo gestionarse por cuenta propia, aprender a partir de la experiencia y adaptarse al cambio. Los equipos de software utilizan Scrum para resolver complejos de forma rentable y sostenible.

    Roles en Scrum

    • Product Owner (Propietario del Producto): Responsable de maximizar el valor del producto y gestionar el backlog del producto.
    • Scrum Master: Facilita el proceso de Scrum, elimina obstáculos y asegura que el equipo siga las prácticas de Scrum.
    • Equipo de Desarrollo: Un grupo auto-organizado y multifuncional que trabaja en la creación del incremento del producto.

    Artefactos en Scrum

    • Product Backlog (Backlog del Producto): Lista priorizada de todo el trabajo que se necesita realizar en el producto. Gestionada por el Product Owner.
    • Sprint Backlog (Backlog del Sprint): Lista de tareas que el equipo de desarrollo se compromete a completar durante el sprint. Extraída del product backlog durante la sprint planning.
    • Increment: El conjunto de todos los elementos del backlog del producto completados durante un sprint, sumado al valor de los incrementos de todos los sprints anteriores.

    Ciclo del Sprint:

    • Sprint Planning (Planificación del Sprint):
      • Duración: 2 horas.
      • Objetivo del Sprint: Desarrollar y lanzar una versión básica de la aplicación que permita a los usuarios registrar y visualizar sus transacciones.
      • Historias de Usuario Seleccionadas: Registro de transacciones, pantalla de inicio, autenticación de usuarios.
    • Daily Scrum (Scrum Diario):
      • Duración: 15 minutos cada mañana.
      • Propósito: Cada miembro del equipo comparte lo que hizo ayer, lo que planea hacer hoy y cualquier impedimento que esté enfrentando.
    • Desarrollo:
      • Herramientas: Utilización de un tablero Kanban para visualizar el progreso de las tareas.
      • Prácticas: Desarrollo incremental y pruebas continuas para asegurar la calidad del código.
    • Sprint Review (Revisión del Sprint):
      • Duración: 1 hora.
      • Actividades: El equipo presenta la versión básica de la aplicación a los interesados, quienes prueban las funcionalidades y proporcionan feedback. Los interesados sugieren mejoras y nuevas características basadas en esta primera versión.
    • Sprint Retrospective (Retrospectiva del Sprint):
      • Duración: 1 hora.
      • Actividades: El equipo reflexiona sobre lo que funcionó bien, lo que no funcionó y cómo pueden mejorar. Deciden mejorar la comunicación interna y adoptar una nueva herramienta de automatización de pruebas para el próximo sprint.

    A continuación un ejemplo de ello:

    Ejemplo de ciclo de un Sprint en un equipo de desarrollo de software:

    • Planificación del Sprint: El equipo se reúne y, en colaboración con el Product Owner, selecciona las historias de usuario prioritarias del product backlog que se desarrollarán durante el sprint.
    • Desarrollo y Scrum Diario: Cada día, el equipo realiza una breve reunión diaria para sincronizar esfuerzos y ajustar el plan según sea necesario. Durante el sprint, el equipo trabaja en las historias de usuario seleccionadas.
    • Revisión del Sprint: Al final del sprint, el equipo presenta el trabajo completado a los interesados y recoge feedback.
    • Retrospectiva del Sprint: El equipo se reúne para discutir lo que salió bien, lo que no salió tan bien y cómo pueden mejorar en el próximo sprint.

    Aplicación de SCRUM en una situación real

    Para finalizar con este apartado, se investigó a cerca de la aplicación de un caso real en el que se aplicó exitosamente esta metodología

    Aplicación de Scrum en la construcción de un simulador de Redis

    • Tamaño del equipo: 3 personas
    • Dueño del producto (Product Owner): Investigador
    • Facilitador (Scrum Master): Estudiante con experiencia en Scrum
    • Desarrollador: Estudiante con experiencia en metodologías tradicionales
    • Reuniones diarias: 2 veces a la semana
    • Revisión: Solo al final, cuando el producto (simulador) estuvo terminado
    • Reuniones de Retrospectiva: participaban los dos desarrolladores y era realizada al final de cada sprint

    Resultados

    La selección de un modelo como Scrum ayudó a este grupo a desarrollar este proyecto al adaptarse a las necesidades del mismo.

    Reflexión

    Seleccionar esta metodología puede ser un apoyo debido a la felxibilidad de su naturaleza para así poder diversificar las tareas de los involucrados.
